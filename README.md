# ru-split-amneziavpn
Список российских сайтов и сетевых адресов для раздельного туннелирования в AmneziaVPN.

Идея проекта простая: сервисы, которые должны открываться напрямую, а не через VPN, выносятся в отдельный JSON-файл для импорта и сопровождения.

## Для чего нужен проект

Некоторые российские сайты и сервисы работают нестабильно или ограниченно при подключении через зарубежный VPN.  
Этот репозиторий собирает и поддерживает список адресов, которые лучше исключать из VPN-туннеля.

## Что внутри

- `data/catalog.json` — основной каталог записей с метаданными.
- `data/excluded.json` — спорные, устаревшие или временно отключённые записи.
- `dist/amnezia-sites.json` — готовый JSON для AmneziaVPN.
- `docs/criteria.md` — правила включения и исключения доменов.
- `scripts/build.py` — сборка итогового файла из каталога.

## Формат итогового файла

Финальный файл совместим с простым форматом:

```json
[
  {
    "hostname": "gosuslugi.ru",
    "ip": ""
  },
  {
    "hostname": "nalog.gov.ru",
    "ip": ""
  }
]
```

## Критерии включения

Запись добавляется в список, если выполняется хотя бы одно условие:

- сервис стабильно не открывается через зарубежный VPN;
- сервис требует российский IP полностью или частично;
- сервис использует отдельные поддомены авторизации, оплаты или API, без которых основной сайт работает некорректно;
- проблема воспроизводится более чем у одного пользователя или подтверждается повторной проверкой.

## Критерии исключения

Запись не добавляется или удаляется, если:

- это слишком широкий диапазон без понятной привязки к конкретному сервису;
- домен больше не используется;
- ограничение было временным;
- запись дублирует уже покрываемый адрес;
- нет подтверждения, что проблема связана именно с маршрутизацией через VPN.

## Как использовать

1. Скачайте файл `dist/amnezia-sites.json`.
2. Импортируйте его в список исключений или split tunneling в AmneziaVPN.
3. Проверьте нужные сервисы без VPN-туннеля.

## Как вносить изменения

- Добавляйте новые записи в `data/catalog.json`.
- По возможности указывайте причину добавления и дату проверки.
- Не добавляйте случайные CIDR-диапазоны без объяснения.
- Для спорных записей используйте `data/excluded.json`.

## План развития

- Разделение записей по категориям: госуслуги, банки, маркетплейсы, медиа, транспорт.
- Автоматическая проверка дублей.
- Нормализация доменов и подсетей.
- Автосборка `dist/amnezia-sites.json` через GitHub Actions.

## Отказ от гарантий

Проект поддерживается сообществом и не гарантирует, что каждый адрес будет работать одинаково у всех пользователей.  
Политики доступа, CDN, антибот-защита и геоограничения могут меняться без предупреждения.
